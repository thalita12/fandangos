Modelo SMACSS

SMACSS (Escalável e Arquitetura Modular para CSS) define uma estrutura para organizar o CSS, deixando-o mais flexível e fácil para se manter.

O SMACSS é baseado em categorizações (padrões), que são Base, Layout, Módulo, Estado e Tema.

Regras de Base

Ele é para definir o estilo padrão de como esse elemento deve ser em todas as ocorrências na página.

Utiliza-se um arquivo Reset de CSS para retirar ou redefinir a margem padrão, preenchimento, e outras propriedades.

Regras de Layout

Os principais estilos de layout, como cabeçalho e rodapé são de estilo tradicional usando seletores ID, mas há os elementos que são comuns em todos os componentes da página e para isso é necessário usar os seletores de classes.

Os componentes principais são referidos como estilos de layout.

Regras de Módulos

Cada módulo deve ser criado como um componente independente. Ao fazê-lo, a página será mais flexível.

Se bem feito, os módulos podem ser facilmente deslocados para diferentes partes do layout sem quebrá-los.

Os componentes que pertencem à página são os menu, dropdown, botões, tooltip, foto, etc.

Regras de Estado

Um estado é algo que adiciona e substitui todos os outros estilos.

Os estilos de estado podem aplicar para o layout e/ou estilos de módulos.

E também indicam uma dependência de JavaScript para os efeitos.

Geralmente são mensagens de erros e avisos ou até mesmo de seleção do componente.

Regras de Temas

O tema define as cores e imagens que dão a sua aplicação ou site a sua aparência.

Os temas podem substituir os estilos de base, como as cores dos links padrão.

Nomenclatura AZ

Há uma convenção nos nomes das variáveis e classes para criar.

A nomenclatura dos prefixos: az (refere-se ao que pertence a AZ), fa (refere-se ao que pertence ao fandangos), entre outros.
